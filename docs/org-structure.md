Company Org (Control Station OS)
================================

Executive & Direction
└─ CEO / GM
   ├─ Chief Product Officer (CPO)
   │  └─ Product Management
   │     ├─ Lead PM (Mission Control)
   │     │  ├─ PM (Auth & Identity)
   │     │  ├─ PM (Dashboard & Telemetry)
   │     │  ├─ PM (Focus & Sessions)
   │     │  └─ PM (Desktop/Electron)
   │     └─ Program Management
   │        └─ TPM (Delivery & Planning)
   ├─ CTO / Chief Architect
   │  ├─ Architecture Council
   │  │  ├─ Principal Engineer (Frontend)
   │  │  ├─ Principal Engineer (Backend/API)
   │  │  └─ Principal Engineer (Platform/Infra)
   │  └─ Security & Compliance Lead
   └─ COO / Ops
      └─ Service Operations Manager

Product Design & Research
└─ Head of Design
   ├─ UX Lead
   │  ├─ UX Designer (Auth Flows)

   └─ User Research Lead

Engineering

   │  ├─ Engineering Manager (Frontend)
   │  │  ├─ Senior FE Engineer (Shell/App)
   │  │  ├─ FE Engineer (Auth)
   │  │  ├─ FE Engineer (Dashboard/Telemetry)
   │  │  ├─ FE Engineer (Focus/Game UX)
   │  │  └─ FE Engineer (Electron/Desktop)
   │  └─ FE Platform (Tooling/CI for UI)
   ├─ Backend/API Guild
   │  ├─ Engineering Manager (Backend)
   │  │  ├─ Senior BE Engineer (FastAPI/API)
   │  │  ├─ BE Engineer (Auth & Sessions)
   │  │  ├─ BE Engineer (Telemetry/Storage)
   │  │  └─ BE Engineer (Notifications/Realtime)
   │  └─ Data Engineer (Pipelines/ETL)
   ├─ Infra/SRE
   │  ├─ SRE Lead
   │  │  ├─ SRE (Observability/Monitoring)
   │  │  ├─ SRE (Release/Environments)
   │  │  └─ SRE (Cost & Capacity)
   │  └─ DevOps Engineer (CI/CD)
   ├─ Security/Compliance
   │  ├─ AppSec Lead
   │  ├─ Security Engineer (Auth/Crypto)
   │  ├─ Security Engineer (Infra/Threat)
   │  └─ Compliance Analyst (Policy/Audit)
   └─ Data & ML
      ├─ Head of Data
      ├─ Data Scientist (Behavior/Usage)
      └─ ML Engineer (Ranking/Insights)

Quality & Productivity
└─ Director of QA
   ├─ QA Lead (Frontend/Electron)
   ├─ QA Lead (Backend/API)
   ├─ Test Automation Engineer (Web/Electron)
   ├─ Test Automation Engineer (API/Load)
   └─ Release Manager (RCs/Sign-offs)

Customer & Field
└─ Head of Customer Success
   ├─ Solutions Engineer (Deployments)
   ├─ Support Lead (Tier 1/2)
   ├─ Support Engineer (Tier 3)
   └─ Technical Writer (Runbooks/Docs)

AI Operations (Codex Agents)
└─ AI Ops Manager
   ├─ Agent Orchestrator (Work dispatcher)
   ├─ Domain Agent – Frontend (React/Electron)
   ├─ Domain Agent – Backend (FastAPI/DB)
   ├─ Domain Agent – QA (Vitest/API tests)
   ├─ Domain Agent – Infra (CI/CD/SRE)
   ├─ Domain Agent – Security (AppSec/Threat)
   └─ Compliance Agent (Policy checks)

Notes
- Use this tree to instantiate AI agents per role; scale headcount per lane as needed.
- Each “Lead/Manager” can expand to additional ICs if workload grows.
